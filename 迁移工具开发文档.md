# 博客迁移工具 - 开发文档

## 项目概述

- **博客**: Astro 5 + Fuwari 主题，部署于 `https://ylif4.github.io`
- **仓库**: `D:\Documents\My Online Blog`，GitHub 地址 `git@github.com:YLiF4/YLiF4.github.io.git`
- **源文章目录**: `D:\OSDisk\Blog`
- **目标文章目录**: `src/content/posts/`
- **部署方式**: GitHub Actions（push main → 自动 build + deploy to Pages）

## 迁移工具架构

### 文件

| 文件 | 用途 |
|------|------|
| `scripts/migrate.mjs` | Node.js HTTP 服务器 + 内嵌 HTML GUI（纯原生，无 npm 依赖） |
| `迁移工具.bat` | Windows 双击启动脚本 |
| `package.json` | 包含 `"migrate": "node scripts/migrate.mjs"` 和 `"editor": "node scripts/migrate.mjs"` |

### 技术栈

- **后端**: Node.js 内置 `http` 模块，端口 3456
- **前端**: 纯 HTML/CSS/JS，内嵌在 .mjs 文件中（template literal），支持暗色模式
- **认证**: 无（本地工具，仅 localhost）

### API 路由

| 方法 | 路径 | 功能 |
|------|------|------|
| GET | `/` | 返回完整 HTML GUI 页面 |
| GET | `/api/posts` | 扫描 `D:\OSDisk\Blog`，返回文章列表 JSON |
| POST | `/api/migrate` | 迁移单篇文章：写回源文件（打标记）+ 写入目标目录 |
| POST | `/api/upload` | 一键 Git 上传：`git add -A` → `git commit` → `git push origin main` |
| POST | `/api/shutdown` | 停止 HTTP 服务，`process.exit(0)` |

### GUI 功能

1. **文章卡片列表** — 每篇可展开编辑：标题、日期、分类、标签、描述、slug、草稿、语言
2. **搜索** — 按标题/分类/标签/文件名过滤
3. **排序** — 未分类在前，最近修改在前
4. **批量编辑** — 统一设置分类、标签、草稿状态
5. **预览原文** — 展开查看 Markdown 前 3000 字符
6. **单篇迁移/更新** — 写回源文件 + 目标文件
7. **全部迁移** — 批量处理所有文章
8. **一键上传到 GitHub** — 调用 git push（工具栏按钮 + 右下角浮动按钮）
9. **退出** — 停止服务（右下角红色浮动按钮 + 关闭标签页自动触发 `beforeunload` → `sendBeacon`）

### 文章元数据自动提取规则

| 字段 | 规则 |
|------|------|
| `title` | 从正文第一个 `# ` 标题提取；若无则用文件名 |
| `published` | 文件修改日期（mtime） |
| `tags` | 留空，手动填写 |
| `category` | 留空，手动填写 |
| `draft` | 默认 `false` |
| `lang` | 默认 `''`（使用站点默认 zh_CN） |

### Fuwari Frontmatter 格式

```yaml
---
title: 文章标题
published: 2026-07-10
description: ''
image: ''
tags: [标签1, 标签2]
category: 分类
draft: false
lang: ''
---
```

### 源文件标记机制

- 首次迁移时，`buildMarkdown(post)` 生成完整 frontmatter + 正文，写回源文件
- 二次扫描时，`parseFrontmatter(content)` 识别已有 frontmatter → 显示"已配置"徽章，按钮变为"更新"
- 关闭 GUI 再打开，之前的配置不会丢失

## .bat 启动脚本 - 完整开发历程

### 需求
1. 双击启动，不显示黑色命令行窗口
2. Node 在后台运行
3. 自动打开浏览器到 `http://localhost:3456`
4. 关闭浏览器后自动停止 Node 服务

### 尝试过的方案

#### 方案 1: VBS 脚本（失败）
- 文件名: `迁移工具.vbs`
- 方法: `CreateObject("WScript.Shell").Run`
- 结果: **双击无反应** — Windows 安全策略禁用了 WSH（Windows Script Host）

#### 方案 2: PowerShell Start-Process 隐藏 + start 浏览器（部分成功）
```batch
powershell -WindowStyle Hidden -Command "Start-Process -WindowStyle Hidden -FilePath 'node' -ArgumentList 'scripts\migrate.mjs'"
timeout /t 2 /nobreak >nul
start "" http://localhost:3456
```
- Node 启动成功，但 **浏览器不弹出**
- 原因: PowerShell 隐藏窗口下 `Start-Process http://...` 无法打开浏览器

#### 方案 3: PowerShell + cmd start 浏览器（部分成功）
```batch
powershell -WindowStyle Hidden -Command "Start-Process -WindowStyle Hidden -FilePath 'node' -ArgumentList 'scripts\migrate.mjs'" 2>nul
timeout /t 2 /nobreak >nul
start "" http://localhost:3456
```
- 添加了 `2>nul` 隐藏 PowerShell 的错误输出
- **有时能弹有时不能**，不稳定

#### 方案 4: PowerShell + explorer 浏览器（曾测试可行，后续不稳定）
```batch
@echo off
cd /d "%~dp0"
powershell -WindowStyle Hidden -Command "Start-Process -WindowStyle Hidden -FilePath 'node' -ArgumentList 'scripts\migrate.mjs' -WorkingDirectory '%~dp0'"
timeout /t 4 /nobreak >nul
explorer http://localhost:3456
exit
```
- 关键改进: 添加 `-WorkingDirectory '%~dp0'`，用 `explorer` 替代 `start ""` 打开浏览器
- `explorer` 处理 URL 更可靠，不受隐藏窗口影响

#### 方案 5: PowerShell Start-Process URL + 页面心跳（当前版本）
- `.bat` 先定位 `node.exe`：PATH → Program Files → Codex bundled Node
- 后台启动 `scripts/migrate.mjs`
- 循环检测 `http://localhost:3456` 是否就绪，最多等待 15 秒
- 用 `Start-Process 'http://localhost:3456'` 交给系统默认浏览器打开
- 页面每 2 秒请求 `/api/ping`
- 关闭页面后心跳消失，服务端约 12 秒内自动 `server.close()` 并释放端口
- 右下角“退出程序”按钮会立即请求 `/api/shutdown`

### .bat 脚本当前状态（2026-07-10 修复）

**最新版本** (`D:\Documents\My Online Blog\迁移工具.bat`):
```batch
@echo off
cd /d "%~dp0"

set "URL=http://localhost:3456"
set "NODE_EXE="

for /f "delims=" %%N in ('where node 2^>nul') do (
  if not defined NODE_EXE set "NODE_EXE=%%N"
)
if not defined NODE_EXE if exist "%ProgramFiles%\nodejs\node.exe" set "NODE_EXE=%ProgramFiles%\nodejs\node.exe"
if not defined NODE_EXE if exist "%ProgramFiles(x86)%\nodejs\node.exe" set "NODE_EXE=%ProgramFiles(x86)%\nodejs\node.exe"
if not defined NODE_EXE if exist "%USERPROFILE%\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe" set "NODE_EXE=%USERPROFILE%\.cache\codex-runtimes\codex-primary-runtime\dependencies\node\bin\node.exe"

if not defined NODE_EXE (
  echo [Error] Node.js not found.
  echo Please install Node.js or add node.exe to system PATH.
  pause
  exit /b 1
)

echo Starting server...

:: 后台启动 Node 服务
powershell -NoProfile -ExecutionPolicy Bypass -WindowStyle Hidden -Command "Start-Process -WindowStyle Hidden -FilePath '%NODE_EXE%' -ArgumentList 'scripts\migrate.mjs' -WorkingDirectory '%~dp0'"

:: 等待端口就绪，最多等待 15 秒
set "READY=0"
for /L %%i in (1,1,30) do (
  powershell -NoProfile -ExecutionPolicy Bypass -Command "try{$r=Invoke-WebRequest -UseBasicParsing -Uri '%URL%' -TimeoutSec 1;if($r.StatusCode -eq 200){exit 0}}catch{}exit 1" >nul 2>&1
  if not errorlevel 1 (
    set "READY=1"
    goto :ready
  )
  ping -n 2 127.0.0.1 >nul
)

:ready

:: 用系统默认浏览器打开网页
explorer "%URL%"

exit
```

**关键改进（2026-07-10）**:
- 将换行符从 LF 转换为 **CRLF** — 修复了双击 bat 时 cmd.exe 无法正确解析多行 `for`/`if` 块导致的语法报错
- 用 `explorer "%URL%"` 替代 `powershell Start-Process` 打开浏览器 — 更可靠
- 用 `ping -n 2 127.0.0.1` 替代 `timeout /t 1 /nobreak` 做延时 — 避免 MSYS2 GNU timeout 冲突

### 已知问题

1. **关闭页面后的停服有约 12 秒延迟** — 这是心跳超时机制，避免刷新页面时被 `beforeunload` 误判为关闭。
   
2. **源文件会被修改** — 迁移工具会写回 frontmatter 到源文件，这是"打标记"功能的设计意图。**重要：Typora 的草稿恢复功能（`%APPDATA%\Typora\draftsRecover\`）可以恢复被覆盖的内容**

3. **.bat 文件必须使用 CRLF 换行符** — 如果用编辑器编辑后保存为 LF，会导致 cmd.exe 解析错误。编辑后请用 `unix2dos` 或编辑器设置确保 CRLF。
